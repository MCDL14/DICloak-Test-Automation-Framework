# 飞书接入文档

本文档说明当前 Dicloak 自动化框架的飞书通知接入方式。当前实现以 `core/feishu.py`、`core/result.py` 和 `config/config.yaml` 为准，不再沿用旧项目的独立通知脚本、外部报告结果扫描和旧配置文件结构。

## 当前实现

飞书通知由 `core/feishu.py` 中的 `FeishuNotifier` 负责，运行器 `core/runner.py` 会在以下场景调用：

1. 环境预检失败时发送失败通知。
2. APP 启动或 CDP 连接失败时发送失败通知。
3. 用例执行完成后发送执行总结，是否发送成功总结由配置控制。
4. 后续模块熔断时可发送熔断通知。

执行结果统计由 `core/result.py` 负责。当前统计字段包括：

- `total`：用例总数。
- `passed`：通过数。
- `failed`：断言失败数。
- `errors`：异常数。
- `skipped`：跳过数。
- `pass_rate`：通过率。
- `failures`：失败和异常摘要。

## 配置位置

飞书配置位于 `config/config.yaml` 的 `feishu` 节点。真实配置文件已加入 `.gitignore`，仓库中只保留 `config/config.example.yaml` 作为模板。

```yaml
feishu:
  # 是否启用飞书通知
  enabled: true
  # 飞书自定义机器人 webhook，真实地址不要提交到仓库
  webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxxx"
  # 需要 @ 的飞书用户 open_id；为空则不 @ 人
  at_open_id: ""
  # @ 人显示名称；为空时显示 open_id
  at_name: ""
  # 成功时是否发送执行总结
  notify_on_success: false
  # 失败时是否发送通知
  notify_on_failure: true
  # 飞书 HTTP 请求超时时间，单位秒
  timeout: 10
```

配置原则：

1. `webhook_url` 属于敏感信息，只写在本地 `config/config.yaml`。
2. `enabled: false` 或 `webhook_url` 为空时，框架会跳过通知，不影响测试结果。
3. `notify_on_success: false` 时，全部通过不会发送总结；失败仍会按 `notify_on_failure` 配置发送。
4. `at_open_id` 为空时不追加 @；填写后会在消息末尾追加飞书 @ 标签。

## 消息格式

当前使用飞书自定义机器人的文本消息：

```python
payload = {
    "msg_type": "text",
    "content": {
        "text": text,
    },
}
```

请求头：

```python
headers = {"Content-Type": "application/json; charset=utf-8"}
```

@ 人格式：

```text
<at user_id="open_id">display_name</at>
```

## 执行总结示例

用例执行完成后，框架会基于 `RunResult` 组装总结消息。示例：

```text
Dicloak 自动化执行通过
总数：21
通过：21
失败：0
异常：0
跳过：0
通过率：100.0%
```

如果存在失败或异常，会追加失败摘要：

```text
Dicloak 自动化执行失败
总数：21
通过：20
失败：1
异常：0
跳过：0
通过率：95.24%
失败摘要：
failed: tests.p0.environment_management.test_xxx.TestXxx.test_xxx - AssertionError: ...
```

## 失败通知场景

当前失败通知主要覆盖：

1. 环境预检失败：例如 APP 路径不存在、导入文件不存在、CDP 端口不可用、飞书 webhook 配置异常等。
2. APP 启动或 CDP 连接失败：启动后进程检测失败、CDP 连接超时等。
3. 用例执行结果失败：`failed > 0` 或 `errors > 0`。
4. 后续熔断场景：登录失败、APP 启动失败、CDP 连接失败、模块级基础能力失败等。

飞书通知失败不会覆盖原始测试失败原因，只会记录到日志中。

## 成功与失败判断

发送成功条件：

```python
response.status_code == 200 and response.json().get("code") == 0
```

以下情况视为发送失败，并写入日志：

1. HTTP 状态码不是 `200`。
2. 飞书响应 JSON 中 `code != 0`。
3. 请求超时、连接失败、DNS 失败等网络异常。
4. 响应内容不是合法 JSON。

通知发送失败不会改变自动化运行退出码。

## 调试方式

可在 Python 交互或临时脚本中直接调用：

```python
from pathlib import Path

from core.config import load_config
from core.feishu import FeishuNotifier
from core.logger import setup_logger

config = load_config(Path("config/config.yaml"))
logger = setup_logger(config)
notifier = FeishuNotifier(config, logger)
notifier.send_test()
```

常见排查点：

1. `feishu.enabled` 是否为 `true`。
2. `feishu.webhook_url` 是否为空或过期。
3. 飞书机器人是否启用了关键词、签名或 IP 白名单等安全策略。
4. 当前网络是否能访问 `open.feishu.cn`。
5. 日志中是否存在 `Feishu notification failed` 或 `Feishu notification request error`。

## 与恢复机制的关系

当前框架已经在 `core/result.py` 接入用例前后恢复机制。恢复失败只记录 warning，不会直接发送飞书通知，也不会覆盖原始用例失败原因。

如果某条用例因为弹窗、抽屉、下拉浮层、加载遮罩等状态残留导致失败，飞书通知仍以最终用例失败摘要为准；恢复机制会在下一条用例开始前尽量把 APP 恢复到可操作状态。
